Использование dllimport и dllexport в классах C++

Visual Studio 2012
Другие версии
  • Visual Studio 2010

Только для систем Microsoft

Можно объявить с классами C++ dllimport OR dllexport атрибут. Эти формы подразумевают, что импортирован или экспортирован всего класса. Классы экспортированные в этом случае вызываются экспортируемые классами.

В следующем примере определяется экспортируемые класс. Все свои данные и статические функции-члены экспортированы.

Копировать 
#define DllExport   __declspec( dllexport )

class DllExport C {
   int i;
   virtual int func( void ) { return 1; }
};

Обратите внимание, что точного используйте dllimport и dllexport атрибуты на элементах экспортируемые класса запрещены.

классы dllexport

При объявлении класс dllexportвсе свои функции-члены и статические элементы данных экспортированы. Необходимо предоставить определения всех таких членов в одной программе. В противном случае формируется ошибка компоновщика. Единственное исключение из этого правила применяется к чисто виртуальные функции, для которых не нужно предоставить явные определения. Однако поскольку деструктор абстрактного класса всегда называется деструктором для базового класса, чисто виртуальный деструктор всегда следует предусматривать определение. Обратите внимание, что эти правила одинаковы для nonexportable классов.

При экспорте данных типа класса или функции, возвращающие классы, то убедитесь, чтобы экспортировать класс.

классы dllimport

При объявлении класс dllimportвсе свои функции-члены и статические элементы данных импортируются. В отличие от расширения функциональности dllimport и dllexport в типах nonclass, статические элементы данных нельзя определять определение в одной программе, в которой a dllimport класс определен.

Наследование и экспортируемые классы

Все базовые классы экспортируемые класса, должны быть может быть экспортирован. Если нет, создается предупреждение компилятора. Кроме того, все доступные элементы, которые также классы должны быть может быть экспортирован. Это правило позволяет a dllexport класс, который должен наследоваться от a dllimport класс, и a dllimport класс, который должен наследоваться от a dllexport хотя класс (не рекомендуется использовать последний). Как правило, все который доступен клиенту библиотеки DLL (в соответствии с правилами доступа C++) должны быть частью экспортируемые интерфейса. Это включает элементов личных данных, таких, на которые ссылается встроенных функциях.

Селективные импорт и экспорт элемента

Так как функции-члены и статические данные внутри класса неявно имеют внешний переключатель можно объявить их с dllimport OR dllexport атрибут, если весь класс не будет экспортирован. Если импортирован или экспортирован весь класс, то явное объявление функций-членов и данных dllimport OR dllexport запрещает. При объявлении статический элемент данных в определении класса как dllexportопределение должно происходить в пределах одной программы (например с компоновкой nonclass внешний).

Аналогично можно объявить функции-члены с dllimport OR dllexport атрибуты. В этом случае необходимо обеспечить a dllexport определение в пределах одной программы.

Стояще отметить несколько важных аспектов относительно селективных импорта и экспорта участника:

  • Селективные импорт и экспорт элемента лучше всего используются для предоставления версию экспортированного интерфейса класса, накладывающее более строгое ограничение; иными словами, один для которого можно разработать библиотеку DLL, которая предоставляет меньшее число открытых и закрытых функций, чем язык в противном случае разрешитьTfо мере. Также полезно для точной настройки экспортируемые интерфейс: если известно, что клиент, по определению, не может получить доступ к конфиденциальным данным, определенные не нужно экспортировать всего класса.

  • При экспорте одну виртуальную функцию в классе, необходимо экспортировать все они или хотя бы предоставить версии, которые клиент может использовать напрямую.

  • Если имеется класс, в котором используется селективные импорт и экспорт член с виртуальными функциями, функции должны находиться в экспортируемые интерфейсе или указанном встроенной (видимых клиенту).

  • Если указать член как dllexport но не включайте его в определении класса ошибка компилятора. Необходимо указать элемента в заголовке класса.

  • Хотя определение членов класса как dllimport OR dllexport разрешает не удалось переопределить интерфейс, указанного в определении класса.

  • При указании функции-члена в месте, отличном от тела определения класса, в котором он объявлен, предупреждение создается, если функция определяется как dllexport OR dllimport (если это определение отличается от, указанное в объявлении класса).
